# 세미콜론(Semicolon) 개발자 포럼 구현 논의 내용

## 1. 프로젝트 개요 및 기술 스택 선정

### 논의 배경
- 개발자들이 질문과 답변을 공유할 수 있는 커뮤니티 플랫폼 개발 필요성 확인
- Stack Overflow와 같은 Q&A 기반 포럼 형태로 구상

### 기술 스택 선정 논의

#### 프론트엔드
**논의된 옵션들:**
- React vs Vue.js vs SvelteKit
- TypeScript vs JavaScript

**최종 선택 및 이유:**
- **SvelteKit**: 빠른 개발 속도, 적은 번들 크기, 직관적인 문법
- **JavaScript + JSDoc**: TypeScript의 복잡성 대신 JSDoc으로 타입 안정성 확보
- **커스텀 CSS**: Tailwind 대신 직접 작성하여 번들 크기 최적화

#### 백엔드
**논의된 옵션들:**
- Node.js (Express/Fastify) vs Python (FastAPI/Django) vs Go

**최종 선택 및 이유:**
- **Python FastAPI**: 빠른 개발, 자동 API 문서 생성, 강력한 타입 �힌트
- **SQLAlchemy**: ORM을 통한 데이터베이스 관리 효율성
- **SQLite/PostgreSQL**: 개발/운영 환경 분리

## 2. UI/UX 설계 및 컴포넌트 구조 논의

### 디자인 시스템 논의
- **Figma 디자인**: 기존 디자인을 참고하여 일관성 있는 UI 구현
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 대응
- **폰트 선택**: Pretendard 폰트로 한국어 가독성 최적화

### 주요 컴포넌트 설계
**공통 UI 컴포넌트:**
- Button, Input, Card, Badge 등 재사용 가능한 기본 컴포넌트
- 일관된 스타일 가이드 적용

**페이지별 컴포넌트:**
- Header: 네비게이션, 검색, 사용자 메뉴
- Sidebar: 카테고리 필터링, 태그 검색
- QuestionList: 질문 목록 표시 및 페이지네이션
- QuestionDetail: 질문 상세 보기 및 답변 관리
- LoginPage/RegisterPage: 사용자 인증

## 3. 데이터베이스 설계 논의

### ERD 설계
**주요 엔티티:**
- User (사용자): 인증 정보, 프로필
- Question (질문): 제목, 내용, 작성자, 조회수
- Answer (답변): 내용, 작성자, 채택 여부
- Tag (태그): 카테고리 분류
- QuestionTag (질문-태그 관계): 다대다 관계 테이블

### 데이터 관계 설계
- User ↔ Question: 1:N (한 사용자가 여러 질문 작성)
- User ↔ Answer: 1:N (한 사용자가 여러 답변 작성)
- Question ↔ Answer: 1:N (한 질문에 여러 답변)
- Question ↔ Tag: N:M (질문과 태그 다대다 관계)

## 4. API 설계 및 인증 체계 논의

### RESTful API 설계
**인증 관련:**
- POST /api/v1/auth/register: 회원가입
- POST /api/v1/auth/token: 로그인 (JWT 토큰 발급)

**질문 관련:**
- GET /api/v1/questions/: 질문 목록 조회
- POST /api/v1/questions/: 질문 작성
- GET /api/v1/questions/{id}: 질문 상세 조회
- PUT /api/v1/questions/{id}: 질문 수정

**답변 관련:**
- POST /api/v1/answers/: 답변 작성
- PUT /api/v1/answers/{id}: 답변 수정
- DELETE /api/v1/answers/{id}: 답변 삭제

### 보안 및 인증
- **JWT 토큰**: 상태 비저장 인증 방식
- **패스워드 해싱**: bcrypt를 사용한 안전한 패스워드 저장
- **CORS 설정**: 프론트엔드와 백엔드 간 안전한 통신

## 5. 개발 환경 및 도구 설정 논의

### 개발 환경 구성
**프론트엔드:**
- Vite: 빠른 개발 서버 및 빌드 도구
- ESLint + Prettier: 코드 품질 및 일관성 유지
- PostCSS: CSS 전처리

**백엔드:**
- Uvicorn: ASGI 서버
- Pytest: 테스트 프레임워크
- Alembic: 데이터베이스 마이그레이션 (향후 적용)

### 프로젝트 구조 논의
```
semicolon/
├── frontend/                 # SvelteKit 프론트엔드
│   ├── src/
│   │   ├── routes/          # 페이지 라우팅
│   │   ├── lib/components/  # 재사용 컴포넌트
│   │   └── app.css         # 글로벌 스타일
│   └── package.json
└── backend/                 # FastAPI 백엔드
    ├── app/
    │   ├── api/            # API 엔드포인트
    │   ├── core/           # 설정 및 유틸리티
    │   ├── models/         # 데이터베이스 모델
    │   ├── schemas/        # Pydantic 스키마
    │   └── crud/           # 데이터베이스 연산
    └── pyproject.toml
```
